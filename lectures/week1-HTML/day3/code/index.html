XHTML handbook

Doctypes

Doctypes appear at the very to pof the page. They tell the browser how the html on your page should be interpreted.

xhtml 1.1 doctype

xhtml 1.1 is a form of xml and the strictest doc type. Forcing xml rules on the document. The browser doesn't care.

<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">

Block Elements

Block elements provide the main structure of the page. They consume all the space available to them.

Heading

We represent headings with the code <h1> element. The number represents the importance of the heading. 1 is the most important and 6 is the least.

Divisions

The <div> element is used for parts of the document that can't be semantically described by other html block elements. It is usually used together with an id or class attribute.

Table

A table in html contains 1 or more row and 1 or more column. In the olden days they were used WRONGLY for formatting the page. Tables should only be used to represent tabular data.

Inline Elements

Inline elements exist inside block elements. They do not alter the flow of page.
